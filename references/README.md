# Kovina Reference Documents

These documents are the shared standard for all Kovina ecosystem projects. They define how we build, design, and architect software across every Open* application.

| Document | Purpose |
|---|---|
| `DESIGN_PLAYBOOK.md` | UI/UX design system — craft standards, visual principles, component rules, typography, color |
| `PRODUCT_ARCHITECTURE_PLAYBOOK.md` | Product architecture — data flow, state management, security, deployment patterns |
| `MCP-SERVER-BUILD-GUIDE.md` | MCP server specification — tools, auth, deployment, and agent integration patterns |

## Usage

Every Kovina repository should reference these documents from its own `CLAUDE.md` or `AGENTS.md`. They are the source of truth for design and architecture decisions across the ecosystem.
