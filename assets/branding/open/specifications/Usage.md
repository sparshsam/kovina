# Open Product Family — Usage

## Repository README

Each Open application's README should:
1. Begin with the Open lockup (OPEN + product name) as the hero.
2. Include a reference to Kovina in the footer.
3. Use the standard README template in `templates/readme-template.md`.

## Application Header

The application header should use:
- **KOVINA wordmark** (parent brand) on the far left.
- Navigation links on the right.
- The Open lockup may appear in the page hero, not the header.

## Store Listings

- App name in store: "OpenReader" (not "OPEN Reader").
- App icon: Application-specific (not the Open wordmark).
- Screenshots may include the Open lockup.

## GitHub Metadata

- Repository description should begin with "Open" (e.g., "OpenReader — Native PDF reader...").
- Repository topics should include "kovina-ecosystem".
- Repository should use the standard README template.

## About Dialog / Splash Screen

- Application name: "OpenReader" (lockup optional).
- Version information below the name.
- "Part of the Kovina ecosystem" below version.
- The Open lockup may be used as a decorative element.

## Dark Mode

In dark mode, all Open branding elements must use:
- `#FFFFFF` fill for OPEN wordmark and product name.
- No additional colors or effects.

## Relationship with Application Icons

The Open brand is typography-only. Application icons are outside the scope of this system. Each application defines its own icon independently.

Key principle: **The icon represents the app. The OPEN wordmark represents the family. They serve different purposes and should not be combined into a single mark.**
